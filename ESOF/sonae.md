# Exemplo da SONAE
```
SONAE: Empresa de retalho

>> Grupo continente << 

BIT:
Tecnologia por detras disto

- Arquitetura
- Contrução
- Manuntenção

Arquitetura de retalho:
Produção
Warehouse
Store

>> Problemas de arquitetura << 

- Escalabilidade: Picos de trafego
- Integração
. Segurança: Exemplo - "foram atacados"

>> Solução <<

- Contexto
- MacroRequisitos
- Ver se o beneficio existe ou não
- Desenho de processos 
- HAS: O que já tem
- Desenho de requisitos
- TO BE: O que queremos que tenha

>> Outros problemas << 

- Compreender problemas dos clientes finais
- Identificar requisitos incomapativeis
- Falar com os stakholders
- Gerir alteracoes de requistos

>> Solution desing << 

- Ter atenção se fornecedores são estaveis ou nao 
- KAPEX: Contrucao 
- OPEX: Manuntenção
- Quando construimos uma solução devemos ter isto em conta

>> Definir requisitos funcionais <<

Prioriadades:
1. MUST
2. SHOULD
3. COULD

# Pensar solução
# Desenvolver
# Adoção do publico

Para que exista adocao do publico 
> A fase um e dois na deve ser rapida
> Para que a fase dois seja rapida: Fazer fase 1 como deve ser
```
